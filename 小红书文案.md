Claude Code 配置 DeepSeek V4 Pro 教程 + dpset 工具分享

今天分享一个 Windows 上 Claude Code 配置 DeepSeek V4 Pro 的完整流程，以及顺手写的一个小工具。

---

1. 安装 Claude Code

打开 PowerShell，一行命令：

irm https://claude.ai/install.ps1 | iex

装完先确保能用官方 API 跑起来：

claude

首次运行会提示登录 Anthropic 账号或用 API Key。这一步的目标是确认 Claude Code 本身安装没有问题。

---

2. 获取 DeepSeek API Key

去 platform.deepseek.com 注册账号，在 API Keys 页面创建一个 key。

DeepSeek 提供了 Anthropic 兼容接口，地址是 https://api.deepseek.com/anthropic

支持的模型：
- deepseek-v4-pro（1M 上下文）
- deepseek-v4-flash（更快更便宜，适合简单任务）

---

3. 把 Claude Code 指向 DeepSeek

Claude Code 通过环境变量来控制后端。要用 DeepSeek V4 Pro，需要设置以下用户环境变量：

ANTHROPIC_BASE_URL → https://api.deepseek.com/anthropic
ANTHROPIC_AUTH_TOKEN → 你的 DeepSeek API Key
ANTHROPIC_MODEL → deepseek-v4-pro[1m]
ANTHROPIC_DEFAULT_OPUS_MODEL → deepseek-v4-pro
ANTHROPIC_DEFAULT_SONNET_MODEL → deepseek-v4-pro
ANTHROPIC_DEFAULT_HAIKU_MODEL → deepseek-v4-flash
CLAUDE_CODE_SUBAGENT_MODEL → deepseek-v4-pro
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC → 1
CLAUDE_CODE_DISABLE_NONSTREAMING_FALLBACK → 1
CLAUDE_CODE_EFFORT_LEVEL → max

传统做法是打开 Windows 系统环境变量对话框，一条一条手动填。切回 Anthropic 的时候再一条一条删掉或改回来。

---

4. dpset：不想手填于是写了个工具

每次手动折腾 10 个变量确实烦。社区有 cc switch 但需要装 Node 依赖，我只想切个模型不想再多维护一个工具链。

dpset 是一个纯 PowerShell 脚本，0 依赖，Windows 自带就能跑。干了四件事：

dpset on — 切到 DeepSeek（首次运行自动备份当前环境变量）
dpset off — 移除 DeepSeek 配置
dpset reset — 从备份恢复原始配置
dpset status — 看一眼当前状态

安装：

git clone https://github.com/powerfulhang/Install-Dpset.git
cd Install-Dpset
powershell -ExecutionPolicy Bypass -File .\Install-Dpset.ps1

安装过程做了三件事：创建 ~/bin/ 目录，写入 dpset.ps1 和 dpset.bat，把 ~/bin 加入 PATH。装完重启终端就行。

使用：

dpset on
（输入 DeepSeek API Key，然后关掉终端重新打开）
claude

现在 Claude Code 跑的就是 DeepSeek V4 Pro 了。

切回 Anthropic 官方 API：

dpset reset
（重启终端）

---

5. 几个设计上的考虑

只动 User 环境变量，不碰 System，改错了不影响系统其他部分。

第一次运行自动备份。dpset on 会把当前所有受管变量存到 ~/.dpset/backup.json，后面可以随时恢复。

API Key 不落盘。每次 dpset on 都是交互式输入，key 不会写进脚本文件。脚本文件可能随手丢进 dotfiles 仓库或者发给别人看，key 绝对不能跟着文件走。

卸载干净。先 dpset reset 恢复环境，然后删掉 ~/bin/dpset.ps1、~/bin/dpset.bat、~/.dpset/ 三个路径就行。

---

6. 注意事项

每次执行 dpset 命令后需要重启终端，Windows 环境变量的变更只在新的进程里生效。

如果用 dpset off 而不是 dpset reset，会移除 DeepSeek 变量但保留备份文件。下次 dpset on 会直接覆盖当前值而不再新建备份。
